## Session 1 — THECOLLECTIVE_AWS01 infrastructure from scratch

### What we worked on

Set up the full deployment infrastructure for THECOLLECTIVE_AWS01 (EC2 at 52.72.211.242) — from an empty VM with nothing installed to running Traefik, Postgres, and Redis. Also configured multi-account GitHub SSH on both dev and corporate laptops.

### What got done

**Infrastructure repo (`jordimo/THECOLLECTIVE_AWS_INFRA`, public)**

- Created `traefik/docker-compose.yml` — Traefik v3.2, pgvector/pg16, Redis 7 Alpine on a shared `traefik-public` Docker network. Postgres/Redis are internal only (no ports exposed to host). Traefik on port 80 + dashboard on 8080. Commit `ea9fd80`.
- Created `traefik/dynamic/middlewares.yml` — placeholder for shared Traefik middlewares.
- Created `traefik/.env.example` — template with `POSTGRES_PASSWORD`.
- Created `project.sh` — registers projects with Traefik using path-based routing (`/project-name` → web, `/project-name/api` → api). Generates dynamic Traefik config files with `stripPrefix` middleware per project. Commands: `add`, `rm`, `ls`.
- Created `setup.sh` — one-time VM setup: checks Docker, copies `.env.example`, creates `traefik-public` network.
- Created `start.sh` / `stop.sh` — start/stop infrastructure via `docker compose up -d` / `down`.
- Created `deploy.sh` — run from corporate laptop over SSH. Supports `./deploy.sh` (infra only), `./deploy.sh <project>` (specific project), `./deploy.sh --all`.
- Created `bootstrap.sh` — for a completely blank Ubuntu EC2: installs Docker CE + Compose plugin + Git, clones repo to `/app/Deployer`, runs setup. Commit `f06688c`.

**Key design decisions vs local-infra**

| Aspect | Local (`local-infra`) | VM (`THECOLLECTIVE_AWS_INFRA`) |
|---|---|---|
| Routing | Host-based (`joann.local`) | Path-based (`/joann`) |
| TLS | mkcert certs, HTTPS | HTTP only (ports 80/443 pending IT) |
| Network | `local-dev` | `traefik-public` |
| DNS | `/etc/hosts` entries | Public IP only |
| Exposed ports | Postgres 5432, Redis 6379 on host | Internal only |
| Middleware | `redirect-to-https` | `stripPrefix` per project |

### Decisions made

- **Repo is public** — contains no secrets (`.env` is gitignored), and making it public avoids needing GitHub auth on the VM. Simplest path.
- **Path-based routing over subdomains** — VM has only an IP, no domain. All projects share the IP and route by path prefix (`/project-name`).
- **No TLS yet** — ports 80/443 pending IT enablement. Traefik config only has a `web` entrypoint on port 80. HTTPS can be added later.
- **Postgres password via `.env` not in repo** — generated with `openssl rand -base64 32`, stored in Bitwarden, manually placed on VM.
- **Corporate GitHub as default on corporate laptop, personal as default on dev laptop** — matches primary use case of each machine, alias for the other.
