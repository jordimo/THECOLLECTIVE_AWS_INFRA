**Issues encountered during migration**

- **Docker Compose network label mismatch**: `docker network create infra` creates a network without Compose labels. When Compose tried to adopt it, it failed with "incorrect label". Fix: disconnect all containers, remove network, let Compose recreate it
- **Old infra `docker compose down` didn't fully stop**: the old compose had env var warnings (missing POSTGRES_PASSWORD in that shell), containers kept running. Fix: explicit `docker stop` + `docker rm`

- **Apps disconnected after network recreation**: when the `infra` network was removed and recreated, running app containers lost their network attachment. Fix: full `docker compose down && up` cycle for each app
